# MTG Quiz

Flashcard quiz for MTG event staff. Pipeline generates questions locally, frontend is static on GitHub Pages.

## Layout
- `docs/` — GitHub Pages root (`index.html` + `questions.json`). Nothing else goes here.
- `planning/` — project plans, dev docs
- `pipeline/` — Rakefile, review server, export logic
- `review/` — local-only review UI
- `data/` — gitignored. MTGJSON sqlite + comprehensive rules text.

## Conventions
- Ruby + Rake. One gem: `sqlite3`. Everything else is stdlib.
- Frontend: vanilla JS, Bulma via CDN. No JS frameworks.
- Questions generated by prompting Claude Code, not via API.
- Nothing reaches `docs/` without explicit human approval through the review UI.
- Minimal dependencies everywhere. If stdlib can do it, use stdlib.
